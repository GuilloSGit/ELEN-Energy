ELEN Energy — Presupuesto de Desarrollo (Software)
Fecha: 2026-02-02
Moneda: USD (impuestos y retenciones no incluidos)

1) Alcance (MVP)
- Scraper (Python): extracción periódica desde fuentes europeas.
- DB Writer (Node.js): Inserción/actualización en PostgreSQL, con una capa básica de saneamiento y normalización de datos que incluya:
    Unificación de campos clave: Conversión de fechas a formato ISO (YYYY-MM-DD), extracción de valores numéricos desde strings, normalización de espacios y caracteres en textos.
    Estructuración mínima en JSONB: Garantizar que atributos críticos para filtros (ej: year, price_amount, currency) se guarden como campos estructurados dentro del JSONB, además del texto crudo.
    Trazabilidad: Almacenar tanto el valor original como el normalizado.
- Base de Datos (PostgreSQL): esquema de catálogo y leads; preparación para particionamiento por categoría/fecha.
- API mínima/Endpoints internos: lectura básica para el Frontend y Back Office.
- Frontend (Web estática o React ligero): Home, Catálogo con filtros básicos, Ficha de producto con CTA “Solicitar cotización”.
- Admin Back Office: ABM de productos, visualización/export de leads.
- DevOps: Contenedorización con Docker/Compose, configuración de entornos (dev/stage/prod) en un proveedor cloud (ej: Hetzner, AWS), pipeline de CI para pruebas básicas y despliegue manual a producción.

> Exclusión por prueba de impacto (pospuesto para siguiente sprint):
Lógica de Curación de Datos (AI Curator): Limpieza y normalización básica de datos extraídos (unificación de formatos de fecha, moneda y medidas; limpieza de texto; clasificación simple de la condición del equipo).

Duración total estimada MVP: 2 meses (8 semanas)
